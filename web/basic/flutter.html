Important concepts

  • Everything you can place in a variable is an object, and every object is an instance of a class. Even numbers, functions, and null are objects. All objects inherit from the Object class.
  • Although Dart is strongly typed, type annotations are optional because Dart can infer types. In the code above, number is inferred to be of type int.
  • Dart supports generic types, like List
  • Dart supports top-level functions (such as main()), as well as functions tied to a class or object (static and instance methods, respectively). You can also create functions within functions (nested or local functions)
  • Similarly, Dart supports top-level variables, as well as variables tied to a class or object (static and instance variables). Instance variables are sometimes known as fields or properties.
  • Unlike Java, Dart doesn’t have the keywords public, protected, and private. If an identifier starts with an underscore (_), it’s private to its library.

个人理解：

  • 没有基础变量和对象的区分。所有变量都指向对象。函数也是一个 Function 类型的对象(类似 JavaScript)。基类都是 Object。
  • Dart 是强类型语言。可以使用int a=0 两种方式定义变量，也可以使用 var a=0 定义，会做类型推导。更像 Swift ，JavaScript 是弱类型语言。
  • Dart 支持 optional 变量，可能为 null，类似 Swift
  • Dart 也有 setter 和 getter 的定义，语法跟 JavaScript 类似
  • 继承使用 extend 实现单继承，但可以使用 mixin 特性来获取别的类的特性
  • 函数和变量的作用域控制。只有 public 和 private ，用下换线区分。不像 Java 和 Swift 使用 private 等关键字定义的更细。
  • 提供了 Generator 和 async、await，类似 ES6，对异步编程更友好
  • 支持泛型，语法跟 Swift 类似
  • 编译使用 JIT 和 AOT 两种方式，可以兼顾开发效率和发布环境的执行效率。

JIT just in time 即时编译 在程序运行中，将热点代码编译成机器码，提高运行效率。代码运行在 Dart 虚拟机中。

AOT Ahead Of time 运行前编译 在程序运行之前，已经编译成对应平台的机器码，不需要在运行中解释编译，就可以直接运行。

FlutterMethodChannel

native 给 Dart 提供的回调，类似 js 桥

native 端：

FlutterMethodChannel* batteryChannel = [FlutterMethodChannel
      methodChannelWithName:@"samples.flutter.io/battery"
            binaryMessenger:controller];
  __weak typeof(self) weakSelf = self;
  [batteryChannel setMethodCallHandler:^(FlutterMethodCall* call,
                                         FlutterResult result) {
    result(@(batteryLevel));
    // result(FlutterMethodNotImplemented);
    // result([FlutterError errorWithCode:@"UNAVAILABLE"
                                   message:@"Battery info unavailable"
}];

Dart 代码:

static const MethodChannel methodChannel =
      MethodChannel('samples.flutter.io/battery');
final int result = await methodChannel.invokeMethod('getBatteryLevel');

FlutterEventChannel

native 端：

FlutterEventSink _eventSink;
FlutterEventChannel* chargingChannel = [FlutterEventChannel
      eventChannelWithName:@"samples.flutter.io/charging"
           binaryMessenger:controller];
[chargingChannel setStreamHandler:self];    // FlutterStreamHandler delegate


实现 FlutterStreamHandler 的代理方法
- (FlutterError*)onListenWithArguments:(id)arguments

_eventSink(@"charging");

UI 渲染

Widget

Flutter在设计的时候就决定了一旦Widget中展示的数据发生变化，就重新构建整个Widget

定义到Widget中的数据一定是不可变的，需要使用final来修饰

StatefulWidget

创建StatefulWidget时必须创建两个类：

  • 一个类继承自 StatefulWidget ，作为 Widget 树的一部分
  • 一个类继承自 State ，用于记录 StatefulWidget 会变化的状态，并且根据状态的变化，构建出新的 Widget
class MyStatefulWidget extends StatefulWidget {
  @override
  State<StatefulWidget> createState() {
    return MyState();
  }
}

class MyState extends State<MyStatefulWidget> {
  @override
  Widget build(BuildContext context) {
    return <构建自己的Widget>;
  }
}
